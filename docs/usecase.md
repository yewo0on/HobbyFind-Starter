| -------- | ----------------------------------------- |

## 3. 주요 단계 및 이벤트 흐름 (Main Steps and Flow of Events)

### UC-01: 루트페이지 탐색

1. 사용자가 웹사이트에 접속한다.
2. 시스템은 루트페이지를 로드한다.
3. Top Bar(로고, 카테고리 필터)를 표시한다.
4. Hero 섹션에 서비스 소개 문구를 표시한다.
5. 메인 영역에 전체 취미 카드 그리드를 표시한다.
6. 사용자는 스크롤하며 취미를 탐색한다.

---

### UC-02: 카테고리 선택

1. 사용자가 Top Bar의 카테고리 Select에서 카테고리를 선택한다.
2. 시스템은 선택된 카테고리의 전용 페이지로 이동한다.
3. 페이지 상단에 카테고리 Hero 섹션(그라데이션 배경, 배지, 제목, 설명)을 표시한다.
4. 해당 카테고리에 속한 취미 카드만 그리드로 표시한다.
5. 사용자는 취미 카드를 탐색한다.

---

## 4. 대체 흐름 및 엣지 케이스 (Alternative Flows and Edge Cases)

| 상황                  | 처리 방식                    |
| ------------------- | ------------------------ |
| 존재하지 않는 카테고리 URL 접근 | 기본 오류 화면 또는 Home으로 이동    |
| 해당 카테고리에 취미가 없는 경우  | “표시할 취미가 없습니다” 메시지 표시    |
| 데이터 로딩 지연           | 로딩 상태(UI 스켈레톤 또는 텍스트) 표시 |

---

## 6. 비즈니스 규칙 및 제약사항 (Business Rules and Constraints)

* 로그인 기능 없이 모든 주요 기능 이용 가능
* 취미는 반드시 하나의 카테고리(운동형·지능형·예술형)에만 속함
* 카테고리 종류는 3개로 고정
* 사용자 입력 데이터 저장 없음

---

## 7. 예외 처리 절차 (Exception Handling Procedures)

| 예외 상황     | 처리 절차                |
| --------- | -------------------- |
| 서버 오류     | 오류 메시지 표시            |
| 데이터 로딩 실패 | 빈 상태 UI 또는 재시도 안내 표시 |
| 페이지 로드 실패 | 기본 오류 페이지 표시         |

---

## 8. UI 고려사항 (User Interface Considerations)

* 모든 페이지에 공통 Top Bar 유지
* 현재 선택된 카테고리는 시각적으로 강조(active 상태)
* 취미 카드는 그리드 형태로 일관되게 배치
* 반응형 레이아웃 지원(모바일/데스크톱)

---

## 9. 데이터 요구사항 및 데이터 흐름 (Data Requirements and Data Flow)

### 데이터 요구사항

| 항목        | 설명              |
| --------- | --------------- |
| hobbyId   | 취미 고유 식별자       |
| hobbyName | 취미 이름           |
| category  | 운동형 / 지능형 / 예술형 |

### 데이터 흐름

```
시스템 → 취미 데이터 로드
  → 루트페이지: 전체 취미 표시
  → 카테고리 페이지: 선택 카테고리 기준 필터링 후 표시
```

---

## 10. 보안 및 개인정보 고려사항 (Security and Privacy Considerations)

* 로그인 및 회원정보 기능 없음
* 개인정보 수집 및 저장 없음
* 쿠키, 세션 미사용
* URL은 카테고리 식별 목적에 한해 사용
