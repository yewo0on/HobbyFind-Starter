# PRD — HobbyFind

## 1. 제품 상세 설명 (Detailed Product Description)

### 1.1 서비스 개요

| 항목       | 내용                                                         |
| ---------- | ------------------------------------------------------------ |
| 서비스명   | HobbyFind                                                    |
| 서비스 타입 | 웹 서비스                                                      |
| 이용 조건  | 비로그인 사용자 이용 가능                                           |
| 목적       | 사용자가 취미를 카테고리별로 탐색·비교하고, 자신에게 맞는 취미를 빠르게 찾아볼 수 있도록 돕는 정보 탐색 서비스 |

### 1.2 핵심 목표

* 사용자가 **운동형 / 지능형 / 예술형**으로 분류된 취미를 **시각적으로 직관적**으로 탐색할 수 있도록 제공
* 회원가입·로그인 없이 **즉시 취미 탐색이 가능한 구조** 제공
* 명확한 정보 구조와 단순한 UI를 통해 **취미 선택에 대한 진입 장벽 최소화**
* PC·모바일 환경 모두에서 보기 편한 **반응형 그리드 레이아웃** 제공

## 3. 핵심 기능 및 사양 (Core Features and Specifications)

### 3.1 전체 페이지 구조

| 페이지                                      | URL                                  | 구성 요소                          | 설명                    |
| ------------------------------------------- | ------------------------------------ | ---------------------------------- | ----------------------- |
| 루트 페이지                                  | `/`                                  | Top Bar / Hero / 취미 카드 그리드    | 전체 18종 취미 탐색 진입점       |
| 운동형 카테고리 페이지                        | `/sports`                            | 카테고리 Hero 섹션 / 취미 카드 그리드       | 운동형 취미 6종만 필터링된 리스트 |
| 지능형 카테고리 페이지                        | `/intellectual`                      | 카테고리 Hero 섹션 / 취미 카드 그리드       | 지능형 취미 6종만 필터링된 리스트 |
| 예술형 카테고리 페이지                        | `/art`                               | 카테고리 Hero 섹션 / 취미 카드 그리드       | 예술형 취미 6종만 필터링된 리스트 |

* 모든 페이지는 **비로그인 상태에서 바로 접근 가능**해야 한다.
* 카테고리 페이지는 루트 페이지와 동일한 레이아웃을 사용하되, **Hero 섹션과 카드 목록만 해당 카테고리에 맞게 필터링**한다.

---

### 3.2 Top Bar 사양

| 요소         | 설명                                              |
| ------------ | ------------------------------------------------- |
| 로고 영역     | 서비스명 또는 심플한 로고 텍스트, 클릭 시 루트 페이지(`/`)로 이동 |
| 카테고리 필터 | Select 드롭다운 형태로 `운동형`, `지능형`, `예술형` 3개 하위 항목 제공 |
| 동작         | Select에서 항목 선택 시 해당 카테고리 페이지로 이동 (`/sports`, `/intellectual`, `/art`) |

**요구사항**

* 항상 화면 상단에 고정되어 사용자가 어느 페이지에서든 카테고리를 전환할 수 있어야 한다.
* Select 드롭다운 형태로 구현되며, 현재 활성화된 카테고리는 Select에 표시된다.
* 루트 페이지에서는 "카테고리"로 표시되고, 카테고리 페이지에서는 해당 카테고리명이 표시된다.

---

### 3.3 Hero 섹션 사양 (루트 페이지 전용)

| 항목      | 내용                                          |
| --------- | --------------------------------------------- |
| 목적      | 서비스 성격과 가치를 한 문장으로 전달               |
| 구성      | 배지 + 메인 카피 + 서브 설명 + 카테고리 배지 + CTA 버튼 |
| 배지 문구 | "오늘부터 시작하는 나만의 이야기"                      |
| 메인 카피 | "평범한 일상에 색을 더하다" (일부 텍스트 그라데이션 적용)     |
| 서브 설명 | "몸을 움직이는 즐거움, 머리를 쓰는 재미, 마음을 표현하는 기쁨. 당신의 라이프스타일에 딱 맞는 취미를 만나보세요!" |
| 애니메이션 | framer-motion을 활용한 fade in 애니메이션 적용      |

**요구사항**

* Hero 섹션은 루트 페이지 상단(Top Bar 바로 아래)에 위치한다.
* 중앙 정렬로 구성되며, 그라데이션 배경과 애니메이션 블러 효과를 포함한다.
* 카테고리 페이지에서는 Hero 대신 **카테고리 Hero 섹션**이 노출된다.

---

### 3.4 카테고리 Hero 섹션 사양 (카테고리 페이지 전용)

| 항목     | 내용                                           |
| -------- | ---------------------------------------------- |
| 구조     | 카테고리별 고유 색상 테마를 가진 Hero 섹션                    |
| 배경     | 카테고리별 파스텔 톤 배경 (운동형: `bg-sports-50`, 지능형: `bg-intellectual-50`, 예술형: `bg-art-50`) |
| 애니메이션 | 펄스 애니메이션 블러 효과 (각 카테고리 계열의 연한 색상: `*-200`)                  |
| 배지     | 카테고리 아이콘 + 서브타이틀 (예: "활기찬 하루를 시작하세요") - 카테고리 메인 컬러 배경 (`bg-sports-500` 등)      |
| 제목     | 선택된 카테고리명 (예: "운동형 취미") - 카테고리별 텍스트 색상 (`text-sports-600` 등)      |
| 서브 카피 | 해당 카테고리 특성을 설명하는 1줄 텍스트 (textSecondary)                    |

카테고리별 구성 예시:

* **운동형**: 
  - 배지: "활기찬 하루를 시작하세요"
  - 제목: "운동형 취미"
  - 설명: "몸을 움직이며 에너지를 발산하고, 건강한 라이프스타일을 만들어가세요."
  - 색상: sports 팔레트 기반 (배경: `bg-sports-50`, 블러: `bg-sports-200`, 배지: `bg-sports-500`, 제목: `text-sports-600`)

* **지능형**: 
  - 배지: "새로운 지식의 세계로"
  - 제목: "지능형 취미"
  - 설명: "집중력과 사고력을 기르며, 평생 가는 지적 자산을 쌓아가세요."
  - 색상: intellectual 팔레트 기반 (배경: `bg-intellectual-50`, 블러: `bg-intellectual-200`, 배지: `bg-intellectual-500`, 제목: `text-intellectual-600`)

* **예술형**: 
  - 배지: "창의력을 만개시키세요"
  - 제목: "예술형 취미"
  - 설명: "감성을 표현하고 아름다움을 만들어내며, 나만의 예술 세계를 탐험하세요."
  - 색상: art 팔레트 기반 (배경: `bg-art-50`, 블러: `bg-art-200`, 배지: `bg-art-500`, 제목: `text-art-600`)

### 3.5 취미 카드 (Hobby Card) 사양

| 항목      | 내용                                                          |
| --------- | ------------------------------------------------------------- |
| 표시 정보 | 취미명 (예: "조깅/러닝", "요가")                                     |
| 레이아웃  | 반응형 그리드 내 카드 형태 (카드 하나당 한 개 취미)                        |
| 동작      | Hover 시 elevation + 살짝 이동, Click 시 해당 카드 선택에 따른 페이지 전환 |

**요구사항**

* 카드 자체에는 링크, 상세보기, 좋아요 등의 상호작용을 구현하지 않는다.
* 모바일·태블릿·PC 환경에서 컬럼 수만 달라지는 **반응형 그리드 레이아웃**을 적용한다.
* 카드 제목이 두 줄 이상이 되지 않도록 적절한 폰트 크기와 여백을 사용한다.

### 4.1 사용자 페르소나

| 항목 | 내용                                                     |
| ---- | -------------------------------------------------------- |
| 이름 | 김지민                                                  |
| 나이 | 25세                                                    |

| 니즈 | 카테고리별로 취미를 간단히 비교·탐색하고, 빠르게 후보를 좁히고 싶음      |

---

### 4.2 사용자 시나리오

1. 사용자가 브라우저에서 `HobbyFind`에 접속한다. (루트 페이지 `/` 진입)
2. Hero 섹션을 통해 "운동형 / 지능형 / 예술형"으로 취미가 구분되어 있음을 인지한다.
3. Top Bar의 카테고리 Select에서 **예술형**을 선택해 예술형 카테고리 페이지(`/art`)로 이동한다.
4. 예술형 카테고리 Hero 섹션과 함께 예술형 취미 카드 리스트를 그리드로 탐색한다.
5. 관심 있는 취미(예: "도자기 만들기", "정원 가꾸기")를 몇 개 기억하거나 메모한다.
6. 서비스 내 추가 행동 없이도, 사용자는 선택한 취미를 기준으로 외부에서 더 깊이 조사하거나 실제로 시도해 볼 수 있다.

## 5. 기술 스택 및 구현 가이드 (Technical Stack & Implementation Guide)

### 5.1 기술 스택

| 영역       | 기술 선택                                       | 비고                                    |
| ---------- | ----------------------------------------------- | --------------------------------------- |
| 프레임워크 | **Next.js (App Router)**                       | `src/app` 구조 사용                      |
| 언어       | **TypeScript**                                 | 전역적으로 타입 안정성 확보                        |
| 렌더링 방식 | Static Generation(SSG) 또는 단순 Client Component 렌더링 | 모든 데이터가 고정된 상수이므로 서버 통신 불필요       |
| 스타일링   | **Tailwind CSS**                               | 반응형 그리드 및 여백, 타이포그래피 일관성 확보         |
| UI 컴포넌트 | shadcn-ui (선택)                               | 기본 카드/레이아웃 컴포넌트 활용 가능, 필요 시 추가 설치 |

**프로젝트 규칙 반영**

* 모든 페이지·컴포넌트는 `use client`를 사용한 **클라이언트 컴포넌트**로 구현한다.
* `page.tsx`의 `params`는 **Promise 기반 타입**을 사용한다.
* 취미 데이터는 `src/features/...` 또는 `src/constants` 내 상수로 관리하며, API 호출은 사용하지 않는다.

**향후 확장 고려사항**

* 현재 버전은 **고정 상수 기반**으로 동작하여 런타임 데이터 로딩, 서버 통신, API 호출이 발생하지 않는다.
* 따라서 **로딩 상태, 데이터 로딩 지연, 서버 오류, 데이터 로딩 실패** 등의 시나리오는 현재 구현 범위에서 실제로 발생하지 않는다.
* 다만, 향후 API 연동 또는 백엔드 추가 시를 대비하여 Use Case 문서(`usecase.md`)에는 이러한 예외 처리 시나리오를 **미래 확장용 시나리오**로 정의해 두었다.
* 향후 백엔드 연동 시에는 해당 Use Case의 예외 처리 절차를 참고하여 구현할 수 있다.

### 5.2 페이지/컴포넌트 설계 방향 (요약)

* `src/app/page.tsx`  
  * 루트 페이지 구성: Top Bar + Hero + 전체 취미 카드 그리드

  * 각 카테고리 전용 페이지: Top Bar + 카테고리 Hero 섹션 + 해당 카테고리 취미 그리드
* 공통 컴포넌트  
  * `TopBar`, `CategoryHero`, `HobbyGrid`, `HobbyCard` 등을 분리하여 재사용성을 높인다.

## 6. 제한사항 (Limitations / Out of Scope)

* 로그인 / 회원가입 / 마이페이지 등 **사용자 계정 기능**은 구현하지 않는다.
* 취미 **상세 페이지**는 구현하지 않는다.
* 즐겨찾기, 추천, 검색, 정렬, 필터(난이도/시간/비용 등) 기능은 구현 범위에서 제외한다.
* 취미 목록 추가·수정·삭제 기능(관리자 페이지 포함)은 구현하지 않는다.
* 소셜 공유, 댓글, 커뮤니티, 알림 등 **소셜 기능**은 포함하지 않는다.
* 본 PRD에 명시되지 않은 기능은 모두 **구현 범위에서 제외**한다.
